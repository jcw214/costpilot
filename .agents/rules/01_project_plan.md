## 1. 프로젝트 배경

### 1.1 비즈니스 맥락

프로젝트 기반으로 운영되는 서비스 기업(SI/SM, 컨설팅, 금융 IT 등)은 다수의 프로젝트를 동시에 수행하며, 각 프로젝트에 투입된 **인력·시간·비용을 정확히 추적**하는 것이 경영 의사결정의 핵심이다. 그러나 현실에서는 다음과 같은 문제가 반복적으로 발생한다:

| 문제 영역 | 현황 | 영향 |
|---|---|---|
| **원가 집계의 분산** | 인건비, 외주비, 공통비용이 엑셀·ERP·그룹웨어 등에 분산 관리 | 프로젝트별 실제 원가 파악 불가, 수작업 집계 시 오류 발생 |
| **공통비 배부 기준 부재** | 관리본부(경영지원, 인사 등) 비용의 배부 기준이 명확하지 않음 | 프로젝트 수익성이 왜곡되어 의사결정에 오류 발생 |
| **실적-예산 괴리 분석 미흡** | 예산과 실적 차이를 사후적·감각적으로만 파악 | 원가 초과의 근본 원인(가격 vs. 효율)을 식별하지 못함 |
| **본부별 성과 평가 어려움** | 매출-원가 데이터가 통합되지 않아 본부 단위 수익성 비교가 불가능 | 조직 개편·자원 재배분 등 전략적 판단 지연 |

### 1.2 왜 "원가/관리회계 시스템"인가?

일반적인 재무회계(Financial Accounting)는 외부 보고 목적의 사후적 기록에 초점을 맞추지만, **관리회계(Management Accounting)** 는 경영자의 **내부 의사결정**을 지원하는 데 목적이 있다. 특히 프로젝트 기반 서비스 기업처럼 **인건비 비중이 높은** 산업에서는:

- **프로젝트별 원가 추적(Job-Order Costing)** 이 생사를 가른다
- 공통 관리비용의 **합리적 배부(Allocation)** 없이는 프로젝트 수익성 판단이 불가능하다
- **표준원가와 실제원가의 차이 분석(Variance Analysis)** 이 원가 통제의 출발점이다

본 시스템은 이러한 관리회계 원리를 실제 동작하는 웹 시스템으로 구현하여, 프로젝트 기반 조직의 데이터 기반 의사결정을 지원하는 관리회계 솔루션이다.

## 2. 프로젝트 목적

### 2.1 핵심 목표

> **"인력, 프로젝트, 본부, 전사 단위의 원가 집계 및 내부대체가액, 표준원가 배분, 원가/성과 요인 분석을 수행하는 시스템을 개발한다. (운영 환경: 5개 본부 구성, 20여 개의 프로젝트 동시 수행)"**

### 2.2 세부 목표

| # | 목표 | 회계 개념 | 시스템 구현 |
|---|---|---|---|
| 1 | **원가 집계** | 직접원가 + 간접원가를 **인력→프로젝트→본부→전사** 4단계로 집계 | 4단계 Drill-Down 원가 집계 대시보드 |
| 2 | **내부대체가액 산정** | 지원본부가 수익본부에 제공하는 서비스의 내부 거래 가액 결정 | 내부대체가액 산정 엔진 및 시뮬레이션 |
| 3 | **표준원가 배분** | 사전 설정된 표준원가를 배부 기준(Cost Driver)에 따라 배분 | 표준원가 배분율 설정 및 배분 결과 조회 |
| 4 | **원가 요인 분석** | 표준원가 vs. 실제원가의 가격차이·능률차이 분해 | 요인별 차이 원인 분석 차트 |
| 5 | **성과 요인 분석** | 본부별 공헌이익, 인력 가동률 등 성과 동인(Driver) 분석 | 본부·프로젝트 수익성 순위 및 성과 요인 대시보드 |

### 3.2 기능 구조 (5개 핵심 비즈니스 모듈)

> **참고**: 아래 모듈 구조는 비즈니스 기능 단위의 논리적 분류이며, 실제 코드 디렉토리 구조와는 별개이다.

```
CostPilot
├── [M1] 원가 집계 모듈 (Cost Aggregation)
│   ├── 인력별 투입 시간·비용 조회
│   ├── 프로젝트별 직접원가 집계
│   ├── 본부별 원가 총괄
│   └── 전사 원가 총괄 (Executive Summary)
│
├── [M2] 내부대체가액 산정 모듈 (Internal Transfer Pricing)
│   ├── 지원본부별 서비스 원가 산출
│   ├── 내부대체가액 산정 방식 선택 (원가기준 / 시장가격기준 / 협의가격)
│   └── 수익본부별 내부대체가액 배부 시뮬레이션
│
├── [M3] 표준원가 배분 모듈 (Standard Cost Allocation)
│   ├── 표준원가 기준 설정 (표준단가 × 표준투입량)
│   ├── 배분 기준(Cost Driver) 선택 (인원수 / 매출 비율 / 투입공수)
│   └── 프로젝트·본부별 표준원가 배분 결과
│
├── [M4] 원가 요인 분석 모듈 (Cost Variance Analysis)
│   ├── 표준원가 대비 실제원가 비교
│   ├── 가격차이(Rate Variance) 요인 분석
│   ├── 능률차이(Efficiency Variance) 요인 분석
│   └── 프로젝트별 예산 소진율 추적
│
└── [M5] 성과 요인 분석 모듈 (Performance Factor Analysis)
    ├── 본부별 공헌이익 계산 및 순위
    ├── 프로젝트 수익성 히트맵
    ├── 인력 가동률(Utilization Rate) 분석
    └── 전사 경영 성과 요약 대시보드
```

### 3.3 각 모듈별 핵심 비즈니스 로직

#### [M1] 원가 집계 — 4단계 집계 체계

```
원가 집계 4단계 Drill-Down 구조:

[Level 1] 인력 단위
    - 개인별 총원가 = Σ(시급 × 투입시간) + 개인 귀속 경비
    - 인력별 프로젝트 투입 내역 추적

[Level 2] 프로젝트 단위
    - 프로젝트별 총원가 = 직접원가 + 배분된 간접원가
    - 직접원가 = 직접인건비 + 직접외주비
    - 직접인건비 = Σ(투입인력i의 시급 × 투입시간i)
    - 직접외주비 = 해당 프로젝트에 직접 귀속되는 외주 계약 비용

[Level 3] 본부 단위
    - 본부별 총원가 = Σ(소속 프로젝트 원가) + 본부 공통비
    - 수익본부: 직접 수행 프로젝트 원가 집계
    - 지원본부: 전사 지원 활동 원가 집계 → 내부대체가액으로 배분

[Level 4] 전사 단위
    - 전사 총원가 = Σ(모든 본부 원가)
    - 전사 매출 대비 원가율, 영업이익률 산출
    - 경영진 Executive Summary 대시보드

간접원가 (지원본부 발생 비용):
  - 경영지원본부 비용: 인사, 총무, 재무팀 인건비 + 운영비
  - 기술인프라본부 비용: 서버 운영비, 보안 라이선스, 공용 개발 인프라
```

#### [M2] 내부대체가액 산정 — 가액 결정 방식

```
내부대체가액(Internal Transfer Price)이란?
  지원본부가 수익본부에 서비스를 제공할 때, 그 서비스에 부여하는
  내부 거래 가액. 이를 통해 지원본부도 독립적 손익을 측정할 수 있다.

가액 산정 방식 3가지:
    ┌──────────────────────────────────────────────────────────┐
    │  산정 방식           │ 산정 기준                          │
    ├──────────────────────┼────────────────────────────────────┤
    │ 원가기준 가액       │ 지원본부 실제 발생원가 그대로 배분    │
    │ 시장가격기준 가액   │ 외부 시장에서 동일 서비스 구매 시 가격 │
    │ 협의가격 가액       │ 원가 + 적정 마진 (지원본부 목표이익)  │
    └──────────────────────────────────────────────────────────┘

산정 예시 (경영지원본부 → 3개 수익본부):
    경영지원본부 총 발생원가: 120,000,000원
    산정 방식: 원가기준 (인원수 비율)
    - 솔루션개발 (30명/60명) → 60,000,000원
    - 시스템운영 (18명/60명) → 36,000,000원
    - 컨설팅     (12명/60명) → 24,000,000원
```

#### [M3] 표준원가 배분 — 배분 프로세스

```
[1단계] 표준원가 설정
    - 표준단가: 직급별 표준 시급 (예: 선임 45,000원/h, 책임 60,000원/h)
    - 표준투입량: 프로젝트 유형별 표준 공수 (예: SI 프로젝트 5,000 M/H)
    - 프로젝트별 표준원가 = 표준단가 × 표준투입량

[2단계] 간접원가 배분 기준(Cost Driver) 선택
    ┌─────────────────────────────────────────────────┐
    │  배분 방식        │ 배분 기준(Driver)              │
    ├───────────────────┼───────────────────────────────┤
    │ 인원수 기준       │ 각 수익본부 소속 인원 비율      │
    │ 매출 비율 기준    │ 각 수익본부 프로젝트 매출 비율   │
    │ 투입공수 기준     │ 각 수익본부 총 투입시간 비율     │
    └─────────────────────────────────────────────────┘

[3단계] 배분 실행
    - 배분액 = Cost Pool × (해당 본부 Driver / 전체 Driver 합계)
    - 배분 결과를 프로젝트별로 다시 세분화 (본부 내 프로젝트 매출 비율)
```

#### [M4] 원가 요인 분석 — 표준원가 차이의 요인 분해

> **이론적 근거**: 한국 공인회계사(CPA) 원가관리회계에서 다루는
> 표준원가 차이분석(Standard Cost Variance Analysis) 체계에 기반한다.

```
총차이 = 실제원가 - 표준원가

[직접노무비 차이 분석] (프로젝트 기반 서비스업의 핵심 원가)
  ┌── 임률차이 (Labor Rate Variance)
  │     = (실제임률 - 표준임률) × 실제작업시간
  │     → 요인: 직급 Mix 변동, 초과근무 수당, 계약 단가 변동
  │
  └── 노무능률차이 (Labor Efficiency Variance)
        = (실제작업시간 - 표준작업시간) × 표준임률
        → 요인: 생산성 저하, 재작업, 요구사항 변경, 기술 난이도

[간접원가(제조간접비) 차이 분석]
  ┌── 예산차이 (Spending Variance)
  │     = 실제간접원가 - 변동예산 허용액
  │     → 요인: 예산 초과 지출, 비용 통제 실패
  │
  ├── 능률차이 (Overhead Efficiency Variance)
  │     = (실제시간 - 표준허용시간) × 변동간접비 배부율
  │     → 요인: 투입 시간이 표준 대비 초과/절감
  │
  └── 조업도차이 (Volume Variance)
        = (실제시간 - 정상조업도) × 고정간접비 배부율
        → 요인: 인력 유휴, 프로젝트 지연, 계절적 수요 변동

판정 기준:
  유리한 차이(F, Favorable): 실제 < 표준 → 원가 절감
  불리한 차이(U, Unfavorable): 실제 > 표준 → 원가 초과

요인 분석 예시 (프로젝트 A):
  총차이: +15,000,000원 (U)
  ├── 임률차이:     +3,000,000원 (U) ← 고급 인력 투입 비율 증가
  ├── 노무능률차이: +8,000,000원 (U) ← 요구사항 변경으로 재작업 발생
  └── 간접원가차이: +4,000,000원 (U) ← 서버 증설비 예산 초과
```

#### [M5] 성과 요인 분석 — 성과 동인(Driver) 식별

```
[성과 요인 1] 본부별 공헌이익 보고서
───────────────────────────────────────
                        솔루션개발  시스템운영  컨설팅    전사합계
매출                     500M       400M      300M     1,200M
(-) 직접원가              200M       180M      130M      510M
─────────────────────────────────────── 
= 직접 공헌이익           300M       220M      170M      690M
(-) 내부대체가액           80M        72M       63M      215M
───────────────────────────────────────
= 본부 공헌이익           220M       148M      107M      475M
(공헌이익률)             (44.0%)    (37.0%)   (35.7%)   (39.6%)
───────────────────────────────────────

[성과 요인 2] 인력 가동률 (Utilization Rate)
  가동률 = 수익 프로젝트 투입시간 / 총 가용시간 × 100%
  - 목표: 80% 이상
  - 80% 미만: 벤치(Bench) 인력으로 분류 → 비효율 요인
  - 본부별/직급별 가동률 비교 → 인력 재배치 의사결정 지원

[성과 요인 3] 프로젝트 수익성 구간 분석
  - A등급 (이익률 40%↑): 우수 프로젝트 → 확대 검토
  - B등급 (이익률 20~40%): 보통 → 원가 개선 여지 분석
  - C등급 (이익률 20%↓): 주의 → 원가 초과 요인 집중 분석
```

```
┌───────────────────▼─────────────────────────────┐
│           Frontend (Next.js + Chart.js)          │
│  - SSR 대시보드 페이지                            │
│  - 인터랙티브 차트 (Bar, Pie, Heatmap)             │
│  - BFF 패턴: API 프록시 + 세션 관리                │
└───────────────────┬─────────────────────────────┘
                    │ HTTP (Docker 내부 네트워크)
┌───────────────────▼─────────────────────────────┐
│           Backend (Spring Boot + JPA)             │
│  - REST API (원가 집계/배부/분석)                   │
│  - 비즈니스 로직 계층 (배부 엔진, 차이 분석 엔진)     │
│  - Mock Data 코드 시딩 (DataInitializer)           │
└───────────────────┬─────────────────────────────┘
                    │ JDBC (Docker 내부 네트워크)
┌───────────────────▼─────────────────────────────┐
│           PostgreSQL 17                           │
│  - 본부/프로젝트/인력/공수/원가 테이블               │
│  - 표준원가, 예산 테이블                            │
│  - 배부 결과 저장                                  │
└─────────────────────────────────────────────────┘
```

### 4.2 기술 스택

| 계층 | 기술 | 선택 근거 |
|---|---|---|
| **Frontend** | Next.js 14 (TypeScript) | 기존 배포 인프라 재활용, SSR, BFF 패턴 |
| **차트** | Recharts (React 네이티브 차트 라이브러리) | 아래 상세 참조 |
| **Backend** | Spring Boot 3.x (Java 21) | 기존 배포 인프라 재활용, JPA 기반 빠른 CRUD 구현 |
| **DB** | PostgreSQL 17 | 기존 인프라 유지, 안정적인 관계형 DB |
| **배포** | Docker Compose + GitHub Actions (self-hosted) | 기존 CI/CD 파이프라인 100% 재활용 |

#### 차트 라이브러리 상세: Recharts

**선택 근거**: React 컴포넌트 기반으로 Next.js와 자연스럽게 통합되며, 선언적 API로 빠른 프로토타이핑이 가능하다. D3.js 기반이지만 D3를 직접 다룰 필요 없이 `<BarChart>`, `<PieChart>` 등의 컴포넌트를 조합하여 구현한다.

| 모듈 | 차트 타입 | 시각화 대상 |
|---|---|---|
| **M1** 원가 집계 | **BarChart** (Stacked) | 인력·프로젝트·본부별 원가 구성 (직접인건비/직접외주비/간접원가 적층) |
| **M1** 원가 집계 | **PieChart / Donut** | 전사 원가 구성비율 (비용 항목별 점유율) |
| **M2** 내부대체가액 | **BarChart** (Grouped) | 산정 방식별 배부 금액 비교 (원가기준 vs. 시장가격 vs. 협의가격) |
| **M3** 표준원가 배분 | **BarChart** (Horizontal) | 프로젝트별 표준원가 배분 결과 |
| **M4** 원가 요인 분석 | **BarChart** (Waterfall 스타일) | 표준원가 → 각 차이 요인별 증감 → 실제원가 분해 |
| **M4** 원가 요인 분석 | **BarChart** (Diverging) | 프로젝트별 유리(F)/불리(U) 차이 방향 비교 |
| **M5** 성과 요인 분석 | **BarChart** (Grouped) | 본부별 매출·직접원가·공헌이익 비교 |
| **M5** 성과 요인 분석 | **BarChart** (Horizontal) | 인력 가동률 본부별/직급별 비교 |
| **M5** 성과 요인 분석 | **ScatterChart** 또는 **Treemap** | 프로젝트 수익성 분포 (매출 규모 × 이익률) |

### 4.3 보안 원칙

| 원칙 | 구현 방안 |
|---|---|
| **외부 API 완전 차단** | 외부 API 호출 코드 없음. 모든 데이터는 내부 Mock Data |
| **포트 최소 노출** | Frontend(3033)만 외부 노출. Backend·DB는 Docker 내부 네트워크 |
| **크리덴셜 관리** | GitHub Secrets로만 주입. 코드에 하드코딩 금지 |
| **컨테이너 격리** | `${USER_ID}` prefix로 다른 사용자 컨테이너와 완전 분리 |
| **데이터 격리** | Mock Data만 사용. 실제 고객 데이터 없음 |
