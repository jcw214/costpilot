## 2. 기능 요구사항

### 2.1 [M1] 원가 집계

> **목적**: 인력→프로젝트→본부→전사 4단계 Drill-Down 원가 조회

| ID | 기능 | 핵심 로직 | 차트 |
|---|---|---|---|
| **COST-STAFF** | 인력별 원가 조회 | `인건비 = Σ(투입시간 × 시급)` | 테이블 |
| **COST-PROJECT** | 프로젝트별 원가 집계 | `총원가 = 직접인건비 + 직접외주비 + 배분간접원가` | Stacked Bar |
| **COST-DEPT** | 본부별 원가 총괄 | `본부 총원가 = Σ(소속 프로젝트 원가) + 본부 공통비` | Grouped Bar |
| **COST-TOTAL** | 전사 원가 총괄 | `원가율 = 총원가 / 매출 × 100%` | Pie + KPI 카드 |

**공통 필터**: 조회 기간(월/분기), 본부, 직급

---

### 2.2 [M2] 내부대체가액 산정

> **목적**: 지원본부 서비스의 내부 거래 가액 산정 및 방식별 시뮬레이션

| ID | 기능 | 핵심 로직 | 차트 |
|---|---|---|---|
| **TRANSFER-POOL** | 지원본부 Cost Pool 조회 | `Cost Pool = Σ(인건비 + 운영비)` | Stacked Bar |
| **TRANSFER-SIM** | 산정 방식별 시뮬레이션 | 아래 상세 | Grouped Bar |

**TRANSFER-SIM 산정 방식:**

```
1. 원가기준:   배부액 = Cost Pool × (해당 본부 Driver / 전체 Driver)
2. 시장가격기준: 배부액 = 원가기준 배부액 × 시장가 가산율
3. 협의가격:   배부액 = Cost Pool × (1 + 목표마진율) × Driver 비율
```

**배부 기준(Cost Driver) 선택**: 인원수 비율 / 매출 비율 / 투입공수 비율

---

### 2.3 [M3] 표준원가 배분

> **목적**: 표준원가 기준 설정, 배분 실행, 표준 vs. 실제 비교

| ID | 기능 | 핵심 로직 | 차트 |
|---|---|---|---|
| **STD-BASE** | 표준원가 기준 조회 | 직급별 표준시급 + 유형별 표준공수 테이블 | 테이블 |
| **STD-ALLOC** | 표준원가 배분 결과 | `배분액 = Cost Pool × Driver 비율` | Horizontal Bar |
| **STD-COMPARE** | 표준 vs. 실제 비교 | `차이 = 실제원가 - 표준원가`, `차이율 = 차이 / 표준원가 × 100%` | Grouped Bar |

**배분 기준 선택**: 인원수 / 매출 비율 / 투입공수

---

### 2.4 [M4] 원가 요인 분석

> **목적**: 표준원가 차이를 요인별로 분해하여 원가 초과 원인을 식별한다.  
> **이론 근거**: 한국 CPA 원가관리회계 표준원가 차이분석 체계

| ID | 기능 | 핵심 로직 | 차트 |
|---|---|---|---|
| **VAR-LABOR** | 직접노무비 차이 분석 | 임률차이 + 능률차이 | Waterfall Bar |
| **VAR-OVERHEAD** | 간접원가 차이 분석 | 예산차이 + 능률차이 + 조업도차이 | Waterfall Bar |
| **VAR-BUDGET** | 예산 소진율 추적 | `소진율 = 사용액 / 예산 × 100%` | Horizontal Bar |
| **VAR-SUMMARY** | 차이 종합 비교 | 프로젝트별 총차이 F/U 방향 비교 | Diverging Bar |

**VAR-LABOR 공식:**

```
임률차이 = (실제임률 - 표준임률) × 실제작업시간
능률차이 = (실제작업시간 - 표준허용시간) × 표준임률
총차이 = 임률차이 + 능률차이
```

**VAR-OVERHEAD 공식 (3분법):**

```
예산차이 = 실제간접원가 - (변동간접비 배부율 × 실제시간 + 고정간접비 예산액)
능률차이 = (실제시간 - 표준허용시간) × 변동간접비 배부율
조업도차이 = (실제시간 - 정상조업도) × 고정간접비 배부율
```

**판정**: `실제 > 표준 → 불리(U)` / `실제 < 표준 → 유리(F)`

---

### 2.5 [M5] 성과 요인 분석

> **목적**: 본부·프로젝트 수익성과 인력 가동률 등 성과 동인(Driver)을 분석한다.

| ID | 기능 | 핵심 로직 | 차트 |
|---|---|---|---|
| **PERF-MARGIN** | 본부별 공헌이익 | `공헌이익 = 매출 - 직접원가 - 내부대체가액` | Grouped Bar |
| **PERF-UTIL** | 인력 가동률 분석 | `가동률 = 수익 프로젝트 투입시간 / 총 가용시간 × 100%` | Horizontal Bar |
| **PERF-PROFIT** | 프로젝트 수익성 분포 | `이익률 = (매출 - 총원가) / 매출 × 100%` | Scatter |
| **PERF-EXEC** | 전사 요약 대시보드 | KPI 집계 | KPI 카드 8개 |

**PERF-MARGIN 구조:**

```
매출 - 직접원가 = 직접 공헌이익
직접 공헌이익 - 내부대체가액 = 본부 공헌이익
공헌이익률 = 본부 공헌이익 / 매출 × 100%
```

**PERF-UTIL 업무 규칙**: 가동률 80% 미만 → 벤치(Bench) 인력 분류

**PERF-PROFIT 등급**: A(이익률 40%↑) / B(20~40%) / C(20%↓)

**PERF-EXEC KPI 항목:**

| # | KPI | 산출식 |
|---|---|---|
| ① | 전사 매출 | `Σ(프로젝트 매출)` |
| ② | 전사 총원가 | `Σ(프로젝트 총원가)` |
| ③ | 영업이익률 | `(매출 - 총원가) / 매출 × 100%` |
| ④ | 전사 원가율 | `총원가 / 매출 × 100%` |
| ⑤ | 평균 가동률 | `Σ(개인 가동률) / 인원수` |
| ⑥ | 최우수 본부 | 공헌이익률 1위 본부명 |
| ⑦ | 주의 프로젝트 수 | C등급 프로젝트 개수 |
| ⑧ | 원가 초과 프로젝트 | 불리한 차이 10% 이상인 프로젝트 수 |

## 6. 접수 기준 (Acceptance Criteria)

- [ ] 원가 집계 4단계가 수치적으로 정합한다 (합계 일치)
- [ ] 내부대체가액 3가지 방식의 결과가 상이하게 표시된다
- [ ] 표준원가 배분 결과 합계 = Cost Pool 총액
- [ ] `임률차이 + 능률차이 = 직접노무비 총차이` 성립
- [ ] 유리(F)/불리(U) 판정이 정확하다
- [ ] 공헌이익 전사 합계 = 각 본부 합계
- [ ] 외부 API 호출 코드 없음
- [ ] `docker compose up --build -d` 한 줄 배포 가능
