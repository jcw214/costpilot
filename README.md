# CostPilot

> 프로젝트 기반 서비스 기업을 위한 원가/관리회계 통합 대시보드 시스템

인력, 프로젝트, 본부, 전사 단위의 **원가 집계**, **내부대체가액 산정**, **표준원가 배분**, **원가/성과 요인 분석**을 수행합니다.

## 주요 기능

| 모듈 | 설명 | 데이터 수정 |
|---|---|---|
| **대시보드** | KPI 4종(매출·총원가·이익률·가동률) + 본부별 공헌이익 차트 | — |
| **원가 집계** | 전사→본부→프로젝트→인력 4단계 Drill-Down 원가 조회 | — |
| **내부대체가액** | 가격정책(원가기준/시장가격/협의가격) × 배부기준(인원수/매출액/투입공수) 시뮬레이션 | — |
| **표준원가 배분** | 직급별 표준시급·유형별 표준공수 기반 기대 원가 산출 및 실제 비교 | — |
| **원가 요인 분석** | 직접노무비(임률·능률) / 간접원가(예산·능률·조업도) 3분법 차이 분석 | — |
| **성과 요인 분석** | 본부 수익성, 프로젝트 이익률, 인력 가동률 종합 평가 | — |
| **기준 데이터** | 본부·인력·프로젝트·직급·유형 마스터 데이터 조회 | ✏️ 실제시급, 계약매출, 표준시급 |
| **거래 데이터** | 투입 공수·직접경비·간접경비 거래 내역 조회 | — |
| **설정** | 표준원가 기준표(참조), 가격 정책 관리, 배부 기준 조회 | ✏️ 가격 정책 배율·활성화 |

## 실행 방법

### 로컬 개발

```bash
# 1. 환경변수 설정
cp .env.example .env
# .env 파일의 DB_USERNAME, DB_PASSWORD 등을 수정

# 2. 로컬 개발 모드 (소스 Volume Mount)
docker compose -f docker-compose.local.yml up -d

# 접속: http://localhost:3033
```

### 프로덕션 배포

```bash
docker compose up --build -d
```

## 설계 결정 및 구현 근거

### 데이터 수정(CRUD) 범위

서비스의 모든 데이터에 대해 완전한 CRUD를 제공하지 않고, **분석 결과에 직접 영향을 미치는 핵심 필드**만 인라인 수정을 허용합니다.

| 수정 가능 항목 | 위치 | 영향 범위 |
|---|---|---|
| 직급별 **표준시급** | 기준 데이터 → 직급 탭 | 표준원가 기준, 원가 차이 분석 |
| 직원 **실제시급** | 기준 데이터 → 인력 탭 | 원가 집계, 임률차이 분석 |
| 프로젝트 **계약매출** | 기준 데이터 → 프로젝트 탭 | 이익률, 수익성 분석, 대시보드 KPI |
| 가격 정책 **배율·활성화** | 설정 → 가격 정책 탭 | 내부대체가액 시뮬레이션 |

> **이유:** 엔티티 간 연관관계(Employee → Department/JobGrade, Project → Department/ProjectType)가 깊어, 무분별한 Create/Delete는 데이터 무결성을 해칠 수 있습니다. 현재는 **"수치를 변경하고 분석 결과가 어떻게 바뀌는지 확인하는"** 시뮬레이션 용도에 최적화되어 있습니다.

### 테이블 정렬 및 페이지네이션

모든 목록성 데이터에 공통 `SortableTable` 컴포넌트를 적용하여 **열 헤더 클릭으로 정렬**(오름차순/내림차순/해제 순환)과 **10건 초과 시 자동 페이지네이션**을 제공합니다. 적용된 화면: 성과 요인 분석(3개 탭), 표준원가 배분, 원가 요인 분석(3개 탭), 원가 집계 인력 목록.

### 직급 정렬 — 위계 순서 적용

직급 열 정렬 시 가나다 순이 아니라 **직급 체계 순서**(사원→대리→과장/선임→차장/책임→부장/수석)로 정렬됩니다. `constants.ts`의 `JOB_GRADE_ORDER` 맵과 `gradeOrder()` 헬퍼 함수로 구현하며, 직급 체계가 변경되면 해당 맵만 수정하면 됩니다.

### 표준원가 배분 탭 — '배분 결과' 제거

기존에는 인원수/매출액/투입공수 기준으로 간접비 배분 비율을 변경하는 탭이 있었으나, 데모 데이터 특성상 수익본부가 3개뿐이라 배부 기준 변경에 따른 차이가 체감되지 않아 제거하였습니다. 추후 본부 수가 늘어나거나 실제 데이터가 투입되면 복원을 고려할 수 있습니다.

### 데이터 업로드 — 추후 구현 예정

기준 데이터·거래 데이터 화면의 "데이터 업로드" 버튼은 현재 **안내 모달**만 표시합니다. 실제 파일 파싱·유효성 검증·벌크 INSERT 로직은 다음 단계에서 구현 예정입니다.

### 인앱 매뉴얼 (컨텍스트 도움말)

각 페이지 헤더의 ❓ 아이콘 또는 키보드 `?` 단축키로 해당 화면에 특화된 도움말 패널을 열 수 있습니다. 도움말 콘텐츠는 `lib/help-content.ts`에 정적 관리되며, 실제 페이지 UI의 용어·기능과 정확히 일치하도록 동기화되어 있습니다. 특히 어떤 데이터가 수정 가능한지, 어디서 수정하는지를 명시합니다.
